# SifterSearch Status — April 2026

> **Supersedes the December 2025 status document.** The CORS/streaming issue documented there was resolved. This document reflects the current architecture and operational state.

## Current Architecture

SifterSearch now runs a **three-layer indexing pipeline** across two machines:

| Machine | Role | Hardware |
|---------|------|---------|
| **tower-nas** | App server | Dell Xeon 80-core, 188GB RAM |
| **boss** | AI server | Strix Halo + GPU, 128GB RAM |

Machines communicate via Tailscale. See [server-migration.md](./server-migration.md) for full infrastructure details.

## Three Databases

| Database | Location | Purpose |
|----------|----------|---------|
| `sifter.db` | `~/sifter/siftersearch/data/sifter.db` | Main content + pipeline state |
| `embedding_cache.db` | `~/sifter/siftersearch/data/embedding_cache.db` | Deduplicated 512-dim embeddings |
| `graph.db` | `~/sifter/siftersearch/data/graph.db` | Entity/concept graph |

## Meilisearch Indexes

| Index | Contents |
|-------|---------|
| `paragraphs` | Base text + 512-dim semantic vectors |
| `paragraphs_enhanced` | Objects, context, hypothetical questions, entity arrays |

## Layered Indexing Status

The three-layer indexing pipeline is implemented and tested:

1. **Base layer** — BM25 + 512-dim semantic search in `paragraphs` index. Fully operational.
2. **Object layer** — Entity/concept extraction via vLLM (Qwen3-32B-AWQ on `boss`). Pipeline in place; extraction runs incrementally.
3. **Enrichment layer** — Disambiguation + HyPE questions with prefix-cached prompts. Pipeline in place; runs after object extraction.

## Migration 44

Migration 44 added the layered indexing schema to sifter.db:
- `content_objects` — extracted entity/concept data per paragraph
- `content_enrichment` — disambiguation + hypothetical questions per paragraph
- `pipeline_versions` — version registry with invalidation rules
- `pipeline_jobs` — layer-aware job queue
- `layer_sync_state` — per-layer sync tracking (replaces old `synced`/`enhanced_synced` booleans)

## Key Modules Added

- `api/lib/embedding-cache.js` — Embedding KV cache
- `api/lib/graph-db.js` — Entity graph (separate DB)
- `api/lib/object-extraction.js` — LLM entity extraction
- `api/lib/entity-resolution.js` — Conservative cross-tradition-safe merging
- `api/lib/enrichment-prompts.js` — Deterministic prompt blocks for vLLM prefix caching
- `api/lib/enrichment-runner.js` — Sliding window enrichment
- `api/lib/window-sizer.js` — Dynamic N from KV token budget
- `api/lib/pipeline.js` — Pipeline version registry + invalidation
- `api/lib/pipeline-scheduler.js` — Layer-aware job scheduling
- `api/routes/graph.js` — Graph API endpoints

## Operational Commands

```bash
# SSH to app server
ssh chad@tower-nas

# Check services
pm2 list
pm2 logs siftersearch-worker --lines 50 --nostream

# Deploy update
cd ~/sifter/siftersearch && git pull && pm2 restart siftersearch-api

# Check database scale
sqlite3 data/sifter.db "SELECT COUNT(*) FROM docs WHERE deleted_at IS NULL;"
sqlite3 data/sifter.db "SELECT COUNT(*) FROM content WHERE deleted_at IS NULL;"
sqlite3 data/embedding_cache.db "SELECT COUNT(*) FROM embedding_cache;"

# Check layer sync state
sqlite3 data/sifter.db "SELECT layer, synced, COUNT(*) FROM layer_sync_state GROUP BY layer, synced;"
```
